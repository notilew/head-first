As páginas web são o melhor prato servido na internet. Até aqui, você criou páginas HTML que vivem apenas em seu próprio computador. Você também criou links para páginas que estão apenas em seu próprio computador. Vamos mudar tudo isso. Neste capítulo, encorajamos você a colocar essas páginas web na internet, onde seus amigos , fãs e clientes possam vê-las. Vamos revelar também os segredos sobre a vinculação a outras páginas ao fazer um crack do código h, t, t, p, :, /, /, w, w, w,. Portanto, pegue suas coisas; nossa próxima parada é em Weblândia.

Encontre uma Empresa de Hospedagem

Para colocar sua páginas na Web, você precisa de um servidor que more na web o tempo todo. Sua melhor aposta seria encontrar uma empresa de hospedagem e deixar para ela a preocupação com os detalhes de manter o servidor no ar. Não se preocupe: encontrar uma empresa de hospedagem é bem simples e barato.

Guia de um Minuto sobre Hospedagem

Não conseguiremos dizer aqui tudo o que você precisa saber sobre uma empresa de hospedagem (afinal de contas, este livro é sobre HTML e CSS), mas lhe dar um empurrãozinho na direção certa. Aqui estão alguns recursos a serem considerados durante a procura por uma dessas empresas.

  • Suporte Técnico: A empresa de hospedagem possui um bom sistema para lidar com suas perguntas técnicas? As melhores responderão às suas perguntas muito rapidamente, tanto por telefone como por e-mail.
  • Transferência de Dados: É uma medida de quantas páginas e dados a empresa de hospedagem permitirá que você envie aos seus visitantes durante um determinado mês. A maioria das empresas oferece em seus planos mais básicos quantidades razoáveis de transferência de dados para sites pequenos. Se você estiver criando um site com previsão de muitas visitas, seria melhor olhar com muito cuidado para esse tópico.
  • Backups: A empresa de hospedagem faz backups regulares de sua páginas e dados, garantindo que sejam recuperados no caso de falha de hardware do servidor?
  • Nomes de Domínio: A empresa de hospedagem inclui um nome de domínio em seu preço?
  • Confiabilidade: A maioria das empresas de hospedagem afirma que mantém os sites web no ar por 99% do tempo ou mais.
  • Benefícios: O seu pacote inclui outros benefícios como endereços de e-mail, fóruns de discussão ou suporte para linguagens de script (algo que ser tornará muito importante para você no futuro)?

Olá, meu Nome é Domínio

Mesmo que você nunca tenha ouvido falar de um nome de domínio, já viu e usou zilhões de deles; você sabe... google.com.br, yahoo.com.br, amazon.com, disney.com e talvez alguns que você não gostaria de ver mencionados aqui.

Então, o que é um nome de domínio? Apenas um nome único que é usado para localizar seu site.

Comandos do FTP

Esteja você digitando com os comandos na linha de comando ou usando a aplicação FTP com interface gráfica, os comandos ou operações que podem ser executados são basicamente os mesmos.

  • dir: Obtém uma listagem do diretório atual
  • cd: Muda para o outro diretório. Aqui ".." também significa subir um diretório.
  • pwd: Exibe o diretório em que você está atualmente.
  • put (nome_do_arquivo): Transfere o nome do arquivo especificado para o servidor.
  • get (nome_do_arquivo): Recupera o nome do arquivo especificado do servidor para o seu computador.

Avenida Principal - URL

Antes de mais nada, os endereços web que você digita no browser são chamados de URL's, ou Uniform Resource Locators (Localizadores Uniformes de Recursos). Uma URL é um endereço global composto por: protocolo + web site + path absoluto do recurso, que pode ser usado para localizar qualquer coisa na web incluindo página HTML, áudio, vídeo e muitas outras formas de conteúdo web.

O que é HTTP?

Também conhecido como HyperText Transfer Protocol (Protocolo de Transferência de HiperTexto). É um método acordado (um protocolo) para transferência de documentos de hipertexto pela web. Enquanto que documentos de hipertexto normalmente são apenas páginas HTML, o protocolo também pode ser usado para transferir imagens ou qualquer outro arquivo que venha a ser necessário a uma página HTML.

O HTTP é um protocolo simples de solicitação e resposta.

O que é um Path Absoluto?

Um path absoluto diz ao servidor como fazer o "get" de sua pasta raíz para uma página ou arquivo em particular.
